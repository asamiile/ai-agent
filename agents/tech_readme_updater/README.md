# README.md 自動更新エージェント

このエージェントは、プロジェクト内の `README.md` ファイルを自動的に生成・更新することを目的としています。Google Gemini AI を利用して、プロジェクト全体の概要をまとめたルートの `README.md` および、各エージェントディレクトリ専用の `README.md` を作成します。

### 機能詳細と目的

`tech_readme_updater` エージェントは以下の主要な機能を提供します。

1.  **リポジトリのファイル内容収集**: プロジェクトルート以下の関連するコードファイル (`.py`, `.md`, `.yml`, `.json`, `.txt` 拡張子を持つファイル) の内容を再帰的に読み込み、AIへのプロンプトとして使用するコンテキストを構築します。この際、Gitリポジトリ、仮想環境、キャッシュディレクトリなど、不要なファイルを自動的に除外します。
2.  **ルート `README.md` の生成・更新**: 収集したリポジトリ全体のコンテキストを基に、プロジェクト全体の概要、ディレクトリ構成、各エージェントの役割などをまとめた `README.md` をプロジェクトルートに生成または更新します。
3.  **エージェント別 `README.md` の生成・更新**: `agents/` ディレクトリ配下にある各エージェントフォルダ（例: `agents/fasion_diesel_kids/`）に対し、それぞれのエージェントの機能、実行方法、依存関係などを記述した `README.md` を生成または更新します。
4.  **AI連携**: Google Gemini API (`gemini-2.5-flash` モデル) を使用し、自然言語処理の能力を活用して、与えられたコードベースから適切な README コンテンツを生成します。

このエージェントの目的は、コードベースの変更に伴うドキュメント更新の手間を削減し、常に最新かつ正確なプロジェクトドキュメントを維持することです。

### 実行方法

このエージェントは、プロジェクトのルートディレクトリから以下のコマンドで実行できます。

bash
python agents/tech_readme_updater/main.py


**事前準備:**

1.  **Python環境のセットアップ**: `requirements.txt` に記載されている依存関係をインストールしてください。
    bash
    pip install -r requirements.txt
    
2.  **APIキーの設定**: Google Gemini API を利用するため、`GOOGLE_AI_STUDIO_API_KEY` 環境変数を設定する必要があります。プロジェクトルートに `.env` ファイルを作成し、以下のようにAPIキーを記述してください。
    dotenv
    GOOGLE_AI_STUDIO_API_KEY="YOUR_GEMINI_API_KEY_HERE"
    
    `GOOGLE_AI_STUDIO_API_KEY` が設定されていない場合、このエージェントは実行時にエラーを発生させずに終了します（ローカルでの誤爆防止策）。

### 依存している共通モジュール

このエージェントは、`common/` ディレクトリ内の特定のモジュール（例: スクレイピングやDiscord通知）に**直接的に依存していません**。

しかし、README生成のための入力コンテキストとして、プロジェクト全体（`common/` ディレクトリ内のファイルを含む）のコードを読み込みます。これは、AIがプロジェクト全体の構造や他のエージェント、共通モジュールの役割を理解し、より正確なREADMEを生成するために行われます。

### ファイル構成

`agents/tech_readme_updater/` ディレクトリは以下のファイルで構成されています。


agents/tech_readme_updater/
├── main.py        # READMEの生成と更新をオーケストレーションするメインスクリプト
└── README.md      # このエージェント自身の説明
